1.1 What is Global Operations? top

Global Operations (GO) is an Egoshooter from Barking Dog Studios.

1.2 What is GOSA? top

GOSA is a tool to administer a GO server at runtime and remotely. It shows the current connected players, the teambalance, current map, round time left, and much more. You can send messages, kick/ban players, set the server values and so on.

1.3 What is GOSSIP? top

Global Operations Server-Side Information Parser is another project to help GO-server administrators to do a good job. It's a kind of a GO-Chatter. It's now in Version 0.2b and serkoon is working hard on it.
GOSA (till Version 1.04) does "know" GOSSIP, but in the matter fact that GOSSIP did not exist, it couldn't support all features ;-) Till Version 1.10 it supports most of options GOSSIP will offer. Till version 1.18 GOSSIP is running and full featured by GOSA.
You can get GOSSIP from here.

2.1 What are the System Requirements? top

Tested on the following Operation Systems: Win2k/Win9x/WinXP
It should also run on: WinME/Win95
GOSA does not need much CPU power. It's not tested but it should run on every system then can install Win98. The screen resolution should be set to a minimum of 800x600 to display the main-window properly.

2.2 Who is the author? top

The application was developed by ~BUG~ Falcon.

2.3 How is it programmed? top

It is programmed using Visual Basic 6.

2.4 How do I use it? top

Take a look at the GOSA helpfile, all questions of handling should be solved with it.

2.5 Where to get GOSA? top

Go to the File-Section and download it.
If you download it for the first time, take the newest "GOSA installation-file (maximum)" file, because you can be sure that you will have all needed DLLs (windows-libraries).

2.6 I got an old version of GOSA, what to do? top

Go to the File-Section and download the "GOSA-EXE only" file. Extract it into your old GOSA folder. Now you've the newest version of GOSA.

2.7 What is the "Check For Updates" button good for? top

GOSA will connect to the website and check for the newest version. GOSA will then tell you if you need an update, but it won't download or update itself.

3.1 I can't install GOSA and the error-messages are in german. top

There can be many reasons why you can't install GOSA. Since version 1.11 the setup is completely in english. Please get a new setup-package.

3.2 I started it the first time, how can I connect to a server? top

The first thing you have to do is: Create an account
Settings->Accounts

  • Enter a name for your account, IP, the Adminport and the password (gameport is optional)
  • Press Add. Your account is now available to connect to.

3.3 I disconnected from server, but "Free-Player-Info" didn't reset. top

This is no bug. The reason for this "missing" reset is the fact, that "free-player-info" isn't retrieved over the administrative connection. The other "connection" (to the gameport) will only be established for retrieving the data and is closed right after that.

3.4 I choose "Show Frags"-option but nothing happens! top

The playertable is updated when the next playinfo is requested.

3.5 I can't import the maplist. What's wrong with it? top

Importing a maplist will only work if you have a file, that contains the maps like they are stored in the server-config-file.
e.g.:
"map0" "antarctica"
"map1" "srilanka"
If the file does not have lines with this format, it will fail.
To "load" a mapfile with the GOSA format / you can simply change the maplist-name in the corresponding account.

3.6 I got a file with TeamIDs but how can I import them? top

The file containing the TeamIDs must use the following format per line (without the brackets):
[id-nr]=[team-name]

3.7 What is the admin-port of the GO-server I have to enter into GOSA? top

Windows server
There is no possibility (I know) to remote-administrate a windows server, so there is no admin-port. If you know one, please let me know and send me a mail here.
Linux server
The standard admin-port for Linux-Servers is 29672. You can change it. For further information goto Official Global Operations - file area

3.8 Getting a "component richtx32.ocx missing or corrupted" error. top

If you upgraded to GOSA version 1.20 you need the "richtx32.ocx" because of the changed display-boxes (for chat). Registrate the "richtx32.ocx" yourself or download the "gosa-maxi" archive with version 1.20 or higher.

4.1 What to do if I found a bug? top

If a bug occures please report it to bugs@gosa.bug-clan.de

4.2 Some window-components look displaced, why? top

Due to the different operating systems and system settings (i.e. font-size) few elements could be displaced and/or can overlap and are not properly displayed.
